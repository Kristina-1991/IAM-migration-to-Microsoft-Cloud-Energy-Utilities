# IAM Migration Cutover Plan

**Version:** 1.0
**Cutover Window:** 
**Go-Live Target:** 
**Document Owner:** Business Analyst
**Technical Owner:** IAM Architect

## 1. Cutover scope and waves
| Wave   | Applications                                  | Users       | Authentication Method    | Rollback Capability |
| ------ | --------------------------------------------- | ----------- | ------------------------ | ------------------- |
| Pilot  | 3 P1 apps (ERP Finance, Workday, Critical HR) | 50 users    | Entra ID SSO + Hybrid AD | Full rollback       |
| Wave 1 | 15 P1/P2 apps (SCADA OT, Legal)               | 500 users   | Entra ID + AD Federation | Selective rollback  |
| Wave 2 | 25 P3 apps (CRM, Collaboration)               | 2,000 users | Full Entra ID SSO        | App-level revert    |
| Wave 3 | Remaining apps                                | All users   | Full Entra ID            | Monitoring only     |

## 2. Pre-Cutover checklist (D-7 to D-1) 
| Category    | Activity                         | Owner            | Status | Criteria              |
| ----------- | -------------------------------- | ---------------- | ------ | --------------------- |
| Environment | Entra ID configuration validated | IAM Architect    | ☐      | 100% SSO success rate |
| Data        | Workday → Entra ID sync tested   | IT Ops           | ☐      | <0.1% sync errors     |
| Governance  | Access packages deployed         | Entra Governance | ☐      | 95% coverage          |
| Monitoring  | Sentinel dashboards active       | Security         | ☐      | Real-time alerts      |
| Rollback    | AD fallback tested               | IT Ops           | ☐      | <15 min recovery      |
| Business    | Training completed               | BA               | ☐      | 95% completion rate   |
| Comms       | User notifications sent          | BA               | ☐      | 100% sent             |

## 3. Cutover execution timeline (D-Day)

**D-Day T-24h (Pre-Cutover Freeze)**
08:00 - Final business approval (Steering Committee)
12:00 - Communication "Migration starts tomorrow"
16:00 - All changes frozen (no new requests)

**D-Day (Saturday - Low Impact Window)**
06:00 - Cutover war room activation
07:00 - Wave 1 sync freeze (Workday → Entra ID)
08:00 - Entra Connect hybrid sync
09:00 - SSO configuration activation (Pilot + Wave 1)
10:00 - Access package deployment
11:00 - Sentinel monitoring live
12:00 - Smoke tests (BA validation)
14:00 - Pilot user validation (50 users)
16:00 - Wave 1 go/no-go decision
18:00 - Post-cutover monitoring begins
20:00 - Initial KPI dashboard review

**D+1 to D+7 (Stabilization)**
Daily business KPI reviews, incident triage, rollback if needed

## 4. Go/No-Go Decision Gates 
| Gate         | Criteria                   | Success Threshold   | Decision Authority  |
| ------------ | -------------------------- | ------------------- | ------------------- |
| T-24h        | Pre-cutover checklist 100% | All green           | Steering Committee  |
| Post-Pilot   | SSO ≥98%, incidents <2%    | Pilot success       | BA + IAM Architect  |
| Wave 1       | Business KPIs ≥90%         | Wave 1 approval     | CISO + CIO          |
| Full Go-Live | <2% disruption             | Production approval | Executive Committee |

## 5.Monitoring and KPIs (Real-Time dashboard) 
| KPI Category   | Metric           | Target        | Alert Threshold |
| -------------- | ---------------- | ------------- | --------------- |
| Authentication | SSO Success Rate | ≥98%          | <95% → SEV1     |
| Provisioning   | New Access Time  | <24h          | >48h → SEV2     |
| User Impact    | Helpdesk Calls   | -40% baseline | +20% → SEV2     |
| Compliance     | Access Reviews   | 100% complete | <95% → SEV3     |

## 6. Rollback procedures 

| Scenario         | Rollback Steps                               | R-Time  | Owner          |
| ---------------- | -------------------------------------------- | ------- | -------------- |
| Pilot Failure    | Revert Entra Connect sync, AD authentication | 15 min  | IT Ops         |
| Wave 1 Issue     | Selective app revert to AD                   | 30 min  | IAM Admin      |
| Critical Failure | Full AD fallback + Entra ID suspend          | 2 hours | Emergency Team |
