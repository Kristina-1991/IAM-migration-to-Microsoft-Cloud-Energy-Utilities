# IAM Service Level Agreements (SLAs) - IT Operations Teams

Effective Date: 22/11/2025
Version: 1.0
Owner: IAM Center of Excellence
Review Cycle: Quarterly

## 1. Identity Provisioning SLA

| Process                  | Scope                                   | Target Response Time                                 | Target Resolution Time        | SLA Target | Penalty/Remedy             |
| ------------------------ | --------------------------------------- | ---------------------------------------------------- | ----------------------------- | ---------- | -------------------------- |
| New Hire Provisioning    | Standard employee/contractor onboarding | Request Acknowledged:2 hours
Access Granted:24 hours | Full access package delivered | 95%        | Escalate to IAM CoE        |
| Role Change Provisioning | Internal mover (role/dept change)       | Request Acknowledged:1 hour
Access Updated:8 hours   | Complete role adjustment      | 98%        | Business impact review     |
| Temporary Access         | Short-term project access (<30 days)    | Request Acknowledged:30 min
Access Granted:4 hours   | Access with expiration        | 97%        | Auto-escalation to manager |

## 2. Critical access requests SLA 

| Priority      | Criteria                                              | Response Time | Resolution Time | SLA Target |
| ------------- | ----------------------------------------------------- | ------------- | --------------- | ---------- |
| P1 - Critical | Production outage, regulatory audit, executive access | 15 minutes    | 2 hours         | 99%        |
| P2 - High     | Business critical app outage, compliance deadline     | 30 minutes    | 4 hours         | 98%        |
| P3 - Medium   | Standard business process blocker                     | 1 hour        | 8 hours         | 95%        |

## 3. Immediate Offboarding SLA

| Scenario        | Scope                                | Response Time                                         | Resolution Time              | SLA Target | Verification                |
| --------------- | ------------------------------------ | ----------------------------------------------------- | ---------------------------- | ---------- | --------------------------- |
| Termination     | Employee/contractor termination      | Request Acknowledged: 15 min
Access Revoked:1 hour    | All systems deactivated      | 100%       | Audit trail confirmation    |
| Suspension      | Security incident, performance issue | Request Acknowledged: 5 min
Access Suspended:30 min   | Privileged access revoked    | 100%       | Sentinel alert verification |
| Long-term Leave | Maternity, sabbatical (>30 days)     | Request Acknowledged: 1 hour
Access Adjusted:24 hours | Non-essential access removed | 98%        | Manager sign-off            |


## 4. Incident response SLA 

| Incident Priority | Impact                                     | Initial Response | Resolution Target | SLA Target | Escalation     |
| ----------------- | ------------------------------------------ | ---------------- | ----------------- | ---------- | -------------- |
| SEV 1 - Critical  | Production down >50 users, security breach | 5 minutes        | 4 hours           | 95%        | CISO + CTO     |
| SEV 2 - High      | Production degraded, <50 users impacted    | 15 minutes       | 8 hours           | 97%        | IAM CoE Lead   |
| SEV 3 - Medium    | Single user/app impacted                   | 1 hour           | 24 hours          | 95%        | IT Ops Manager |

## 5. Service availability and performance

| Service              | Availability Target       | Uptime SLA        | Monitoring              |
| -------------------- | ------------------------- | ----------------- | ----------------------- |
| Entra ID SSO         | 99.9%                     | Monthly review    | Sentinel dashboards     |
| Provisioning Service | 99.5%                     | Quarterly audit   | ServiceNow + Entra logs |
| Access Reviews       | Quarterly 100% completion | Governance report | Entra ID Governance     |

## 6. Reporting and review process 
**Monthly SLA Report** (delivered by 5th of following month):

- SLA attainment by process
- Breach analysis & root cause
- Trend analysis & improvement actions

**Quarterly Governance Review:**

- SLA adjustment discussion
- Process improvement roadmap
- Customer satisfaction survey

## 7. Escalation matrix 
Level 1: IT Ops Support → IAM Admin (1 hour)
Level 2: IAM Admin → IAM CoE Lead (2 hours)  
Level 3: IAM CoE → CISO/CTO (4 hours)
Level 4: Executive Steering Committee (24 hours)
